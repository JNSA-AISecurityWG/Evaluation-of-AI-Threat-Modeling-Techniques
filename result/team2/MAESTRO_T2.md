## FactoryQuality
### レイヤー別の脅威
| レイヤー | 構成要素と機能 | 脅威名 | 脅威 | 攻撃シナリオ |
| ---- | ---- | ---- | ---- | ---- |
| 1. 基盤モデル | 自社構築の画像認識モデル。 | 1. サプライチェーン侵害 | T17 Supply Chain Compromise（モデル/ライブラリ供給網侵害） | モデル構築に利用したOSSライブラリに脆弱性があり、推論環境が侵害される。 |
| 2. データ操作 | 訓練画像（Training images）、製品画像（Product images）。 | 2. データ汚染 | T1 Memory Poisoning（文書上はメモリだが、ここでは“学習/蓄積データ汚染”として解釈） | 学習データ画像に悪意のあるデータが混入され、特定の不良品を「良品」と誤認するようにモデルが汚染される。 |
| 3. エージェントFW | 画像を受け取り、モデルに推論を実行させ、結果を返す評価ロジック（API Gateway周辺）。 | 3. 意図・目標操作 | T6 Intent Breaking & Goal Manipulation（意図/目標操作） | 人間の目には見えないノイズを加えた製品を流し、システムの判定結果を意図的に操作する。（敵対的サンプル） |
| 4. デプロイ/インフラ | 工場内に設置されたオンプレミスサーバ、ネットワーク。 | 4. リソース過負荷 | T4 Resource Overload（リソース過負荷） | カメラからの画像送信頻度を意図的に異常な速度に上げ、推論サーバの計算資源（GPU等）をダウンさせる。 |
| 5. 評価/可観測性 | 監査ログ / 推論ログ / モデル版管理 / メトリクス監視、異常検知（誤判定率の変動、処理遅延）。 | 5. 否認・追跡不能 | T8 Repudiation & Untraceability（否認・追跡不能） | 誰がいつモデル更新/結果修正/閲覧したかが追えず、品質事故・不正の原因究明ができない。 |
| 6. セキュリティ/コンプライアンス | 工場内ネットワークのアクセス制御、顧客（作業員）のログイン認証。 | 6. 権限侵害 | T3 Privilege Compromise（権限侵害） | 過権限サービスアカウントや設定ミスにより、結果DBやモデル保管庫へ不正アクセス・改ざんが可能 |
| 7. エコシステム | 外部委託APIはない前提だが、現実の運用依存として 人（Customer/運用者）・カメラ機器・学習用データ供給・更新手順 を含む“周辺エコシステム”として扱う | 7. 人間の操作 | T15 Human Manipulation（人間の操作） | 運用者が“急ぎのモデル更新”などを口実に手順を省略し、検証されていないモデルやデータを投入してしまう |
