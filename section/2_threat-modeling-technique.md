# 脅威モデリング手法について

## MAESTRO
### MAESTROとは
MAESTRO（Multi-Agent Environment, Security, Threat, Risk, and Outcome）は、OWASP GenAI Security Projectによって提案された、マルチエージェントシステム（MAS）に特化した脅威モデリング・フレームワークです。

自律的なエージェントが相互に連携するシステムは、単一のAIモデルよりも複雑な攻撃対象領域（アタックサーフェス）を持ちます。MAESTROは、この複雑さを構造化し、エージェント特有の脆弱性を特定するために設計されました。

### MAESTROの7レイヤー・アーキテクチャ
MAESTROは、システムを以下の7つの論理レイヤーに分解して分析します。各レイヤーで発生しうる固有の脅威を特定することがガイドラインの核心です。

| レイヤー | 名称 | 焦点と主な脅威例 |
| :--- | :--- | :--- |
| **Layer 1** | **Foundation Models** | 基盤モデルの完全性、モデルの盗用、学習データの汚染 |
| **Layer 2** | **Data Operations** | RAG（検索拡張生成）の汚染、ベクターストアの整合性、データ漏洩 |
| **Layer 3** | **Agent Framework** | 自律的な実行ロジック、ワークフローのハイジャック、プロンプトインジェクション |
| **Layer 4** | **Infrastructure** | デプロイ環境の脆弱性、エージェントによるリソース枯渇（DoS） |
| **Layer 5** | **Observability** | モニタリングの回避、ログの改ざん、評価指標の操作 |
| **Layer 6** | **Security & Compliance** | 権限昇格、ガードレールのバイパス、ガバナンス違反 |
| **Layer 7** | **Ecosystem** | 外部ツール・人間・他エージェントとの相互作用における信頼の欠如 |


### MAESTROにおける「脅威のベクトル」
MAESTROでは、エージェント環境に特有の「エージェント・ファクター」を考慮して脅威を評価します。

* **自律性（Autonomy）のリスク:** 人間の承認なしにアクションを実行することによる、予期せぬ連鎖反応。
* **非決定性（Non-Determinism）:** 同じ入力でも出力が変わる特性を悪用した、セキュリティチェックの回避。
* **エージェント間通信（A2A）:** エージェント間での機密情報の漏洩や、悪意のあるエージェントによる他エージェントへの「命令インジェクション」。
* **アイデンティティ管理:** 「どのエージェントが、どの権限で、どのアクションを実行したか」の追跡の困難さ。

### 特徴的な分析視点：クロスレイヤー脅威
MAESTROの最大の利点は、単一のコンポーネントだけでなく、レイヤー間をまたぐ脅威を可視化できる点にあります。

* **信頼の連鎖的崩壊（Cascading Trust Failures）:** 1つのエージェントの侵害が、信頼関係を通じてネットワーク全体の侵害につながる。
* **意図の操作（Intent Manipulation）:** モデル（L1）への攻撃が、フレームワーク（L3）を通じて、外部エコシステム（L7）へ壊滅的な影響を与える。
